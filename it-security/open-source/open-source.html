Open Source:

Der Begriff Open Source ("Offene Quelle") bedeutet, dass
der gesamte Quelltext eines Programmes öffentlich zugänglich ist, und jeder den Code lesen kann.
Jeder, der sich mit der Entsprechenden Programmiersprache auskennt, kann auf
diese Weise sehen, was genau das Programm tut. Auch kann er meistens selbst Fehler beheben und sich von dem Programm vielleicht für
eigene Projekte inspirieren lassen.
Der Code kann meistens auf Seiten wie GitHub oder Sourceforge eingesehen werden.
Auch der Code dieser Webseite kann unter https://github.com/safe-im-web/safe-im-web.github.io angesehen werden.

Warum sicherer?

Open Source Software gilt deshalb als sicherer, da (wie oben schon erwähnt) jeder den Code lesen kann. Das heißt, dass (bei großeren
Projekten) teilweise hunderte Personen pro Tag die Code nach Fehlern und Sicherheitslücken durchkämmen.
Sollte jemand dann einen Fehler finden, kann er direkt eine ausgebesserte Version der Programmes einsenden und schon ist der
Fehler behoben.

Und was ist, wenn die Person den Fehler für sich nutzt?

Bei sehr kleinen Projekten besteht tatsächlich die Gefahr, dass nur eine einzelne Person den Fehler entdeckt und diesen
dann für sich selbst ausnutzt. Allerdings ist die wahrscheinlichkeit, dass dies passiert immernoch um einiges Geringer als bei
Programmen, sie nicht Open Source sind.
